Protocolo BLE

Especificação do protocolo

As mensagens são enviadas por pacotes com um máximo de 512 bytes. Todos os inteiros são unsigned. Todos os inteiros que ocupam mais que 1 byte estão em big endian.

Formato geral das mensagens

  • Tipo de mensagem (1 byte)
  • Dados da mensagem (restantes bytes até ao final do pacote)

Tipos de mensagens

Pedido da lista de tags (app->rpi, id: 0x00)

O leitor responde com um pedaço de lista de tags com todas as tags encontradas ou com erro.

Dados: nenhuns

Exemplo:

00 // Tipo de mensagem

Pedaço da lista de tags (rpi->app, id: 0x01)

Dados:

  • Número pedaços no total (1 byte)
  • Número de pedaços em falta (1 byte)
  • Array de:
    • ID da tag (6 bytes)

Exemplo de um último pedaço de lista com 2 tags:

01 // Tipo de mensagem
03 // 3 pedaços no total
00 // 0 pedaços em falta (este é o último pedaço)

// Lista de tags
AA BB CC DD EE FF // ID da primeira tag
10 20 30 40 50 60 // ID da segunda tag

Exemplo de lista sem tags:

01 // Tipo de mensagem
01 // 1 pedaço no total
00 // 0 pedaços em falta (este é o único pedaço)

Pedido da tag mais próxima (app->rpi, id: 0x03)

O leitor responde com um pedaço de lista de tags com apenas uma tag, ou 0 tags se não forem encontradas tags nenhumas. Também pode responder com erro.

Dados: nenhuns

Exemplo:

03 // Tipo de mensagem

Pedido de estado da placa (app→rpi, id: 0x04)

O leitor responde com uma mensagem de informações da placa ou com uma mensagem de erro indicando o erro de comunicação com a placa.

Dados: nenhuns

Exemplo:

04 // Tipo de mensagem

Informações da placa (rpi→app, id: 0x05)

Dados:

  • Versão do firmware (string de tamanho variável, até ao final da mensagem)

Exemplo:

05 // Tipo de mensagem

// "AS3991 Mini Reader Firmware 1.5.1"
41 53 33 39 39 31 20 4D 69 6E 69 20 52 65 61 64 65 72 20 46 69 72 6D 77 61 72 65 20 31 2E 35 2E 31

Erro (rpi->app, id: 0xFF)

  • Tipo de erro (1 byte)

Tipos de erro

  • 0x00: Outro erro não listado
  • 0x01: Erro de comunicação com a placa ou outro erro de hardware
  • 0x02: Erro de leitura RFID
FF // Tipo de mensagem
02 // Tipo de erro
